> **项目名称**: PDF/PPT Chinese Annotator  
> **版本**: 1.0.0  
> **日期**: 2026-05-30  
> **状态**: 已批准

---

## 1. 项目概述

### 1.1 目标
构建一款桌面端软件，用户导入英文 PDF 或 PPT 文件后，系统通过多智能体协作框架自动为每一页生成详细的中文批注。

### 1.2 核心价值
- 自动化英文文档的中文理解和分析
- 多智能体协作确保批注质量
- 支持两种批注呈现模式（覆盖式/侧边栏）
- 灵活的 LLM 提供商切换（云端/本地）

### 1.3 目标用户
- 需要阅读英文技术文档的中文用户
- 学术研究人员
- 翻译和本地化专业人员

## 3. 功能规格

### 3.1 文件导入模块

#### 3.1.1 支持格式
- **PDF**: 版本 1.0 - 2.0，支持加密 PDF（需密码）
- **PPT**: .pptx 格式，支持 .ppt 转换

#### 3.1.2 导入方式
- 文件选择器对话框
- 拖拽导入（支持单文件和批量）
- 最近文件列表（最多 10 个）

#### 3.1.3 批量处理
- 最多同时处理 20 个文件
- 队列管理，支持暂停/继续
- 进度显示（文件级和页面级）

### 3.2 文档解析模块

#### 3.2.1 PDF 解析
```python
class PDFParser:
    def parse(self, file_path: str) -> Document:
        """
        解析 PDF 文件，返回结构化文档对象
        
        提取内容:
        - 文本内容（保留位置信息）
        - 图片及其位置
        - 表格结构
        - 页面尺寸和布局
        - 元数据（标题、作者等）
        """
```

#### 3.2.2 PPT 解析
```python
class PPTParser:
    def parse(self, file_path: str) -> Document:
        """
        解析 PPT 文件，返回结构化文档对象
        
        提取内容:
        - 幻灯片内容（标题、正文、备注）
        - 形状和文本框
        - 图片和图表
        - 动画和切换效果（仅记录，不处理）
        - 母版和布局信息
        """
```

### 3.3 多智能体协作模块

#### 3.3.1 智能体定义

**翻译员 (Translator)**
```python
translator = Agent(
    role="专业文档翻译员",
    goal="将英文技术文档准确翻译成中文，保留专业术语",
    backstory="""
    你是一位拥有 10 年经验的专业翻译员，精通计算机科学、
    人工智能、软件工程等领域的术语翻译。你的翻译风格准确、
    流畅，善于处理长难句和专业概念。
    """,
    tools=[terminology_tool, context_tool],
    llm=llm_service
)
```

**分析员 (Analyst)**
```python
analyst = Agent(
    role="内容结构分析师",
    goal="深入分析文档内容结构，识别关键概念和逻辑关系",
    backstory="""
    你是一位资深的技术分析师，擅长快速理解复杂文档的结构
    和核心概念。你能够识别文档的主旨、论点、支撑细节，
    以及各部分之间的逻辑关系。
    """,
    tools=[structure_tool, concept_tool],
    llm=llm_service
)
```

**批注员 (Annotator)**
```python
annotator = Agent(
    role="专业批注撰写员",
    goal="基于分析结果撰写详细、有洞察力的中文批注",
    backstory="""
    你是一位专业的技术文档批注专家，擅长用简洁明了的中文
    解释复杂概念。你的批注不仅翻译原文，还提供背景知识、
    术语解释和相关联想，帮助读者深入理解。
    """,
    tools=[annotation_tool, reference_tool],
    llm=llm_service
)
```

**审核员 (Reviewer)**
```python
reviewer = Agent(
    role="质量审核专家",
    goal="确保批注的准确性、完整性和可读性",
    backstory="""
    你是一位严格的质量审核专家，拥有敏锐的语言感知力和
    技术背景。你负责检查批注的准确性、一致性、格式规范，
    并提出改进建议。
    """,
    tools=[quality_tool, consistency_tool],
    llm=llm_service
)
```

#### 3.3.2 任务流程

```python
# 任务 1: 翻译
translate_task = Task(
    description="将页面内容从英文翻译成中文，保留专业术语对照",
    expected_output="中文翻译文本 + 术语表",
    agent=translator
)

# 任务 2: 分析
analyze_task = Task(
    description="分析翻译后内容的结构、关键概念和逻辑关系",
    expected_output="结构化分析报告",
    agent=analyst,
    context=[translate_task]
)

# 任务 3: 批注
annotate_task = Task(
    description="基于分析结果，为每个段落生成详细的中文批注",
    expected_output="格式化的批注内容",
    agent=annotator,
    context=[translate_task, analyze_task]
)

# 任务 4: 审核
review_task = Task(
    description="审核批注质量，确保准确性和可读性",
    expected_output="审核通过的最终批注",
    agent=reviewer,
    context=[annotate_task]
)
```

### 3.4 批注渲染模块

#### 3.4.1 覆盖模式
- 在 PDF 页面上添加注释框（Annotation）
- 支持多种注释样式：高亮、下划线、文本框
- 注释位置自动计算（基于原文位置）
- 可调整注释框大小和位置

#### 3.4.2 侧边栏模式
- 应用右侧显示批注面板
- 按页面组织批注内容
- 支持批注搜索和筛选
- 点击批注跳转到对应位置

#### 3.4.3 模式切换
- 实时切换，无需重新处理
- 保持批注内容同步
- 记住用户偏好设置

### 3.5 导出模块

#### 3.5.1 导出格式
- **带批注 PDF**: 保留原始格式，嵌入批注
- **纯批注文档**: 仅包含批注内容（Markdown/Word）
- **双语对照**: 原文+翻译+批注并排显示

#### 3.5.2 导出选项
- 选择导出页面范围
- 批注样式自定义
- 文件命名规则

### 4.2 核心交互流程

1. **导入文件**
   - 点击"导入"或拖拽文件
   - 显示文件预览和页面数量
   - 确认开始处理

2. **处理过程**
   - 显示进度条（整体 + 当前页面）
   - 实时显示智能体工作状态
   - 支持暂停/取消

3. **查看批注**
   - 左侧文档预览，右侧批注面板
   - 点击批注高亮对应位置
   - 支持批注编辑和删除

4. **导出结果**
   - 选择导出格式和选项
   - 预览导出效果
   - 保存到指定位置

## 5. 配置管理

### 5.1 LLM 配置

```yaml
llm:
  provider: "openai"  # 或 "ollama"
  
  openai:
    api_key: "${OPENAI_API_KEY}"
    model: "gpt-4o"
    temperature: 0.3
    max_tokens: 4096
  
  ollama:
    base_url: "http://localhost:11434"
    model: "llama3:70b"
    temperature: 0.3
```

### 5.2 批注配置

```yaml
annotation:
  mode: "sidebar"  # "overlay" 或 "sidebar"
  detail_level: "detailed"  # "summary", "detailed", "custom"
  
  style:
    font_family: "Microsoft YaHei"
    font_size: 12
    color: "#333333"
    background: "#FFFFCC"
  
  custom_settings:
    include_terminology: true
    include_references: true
    include_examples: true
```

### 5.3 应用配置

```yaml
app:
  language: "zh-CN"
  theme: "system"  # "light", "dark", "system"
  recent_files_limit: 10
  auto_save: true
  temp_dir: "${APPDATA}/pdf-annotator/temp"
```

## 6. 错误处理

### 6.1 错误分类

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 文件格式错误 | 拒绝导入，记录日志 | "不支持的文件格式" |
| 文件损坏 | 尝试修复，失败则提示 | "文件可能已损坏" |
| LLM 连接失败 | 重试 3 次，切换提供商 | "LLM 服务不可用" |
| 处理超时 | 暂停任务，等待用户操作 | "处理时间过长" |
| 内存不足 | 分块处理，减少并发 | "正在优化处理方式" |

### 6.2 恢复机制

- 自动保存处理进度
- 支持断点续传
- 错误日志记录和上报

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 文件导入时间 | < 2 秒 | 100 页 PDF |
| 页面解析时间 | < 1 秒/页 | 不含 LLM 调用 |
